      Annotation    :testNG provides a huge list of annotation which can be useful for deciding the execution of method at what stage.
       			     @BeforeSuite, @AfterSuite, @BeforeClass, @AfterClass, @BeforeTest, @AfterTest, @BeforeMethod, @AfterMethod, @BeforeGroup, @AfterGroup.
      XML Based     :running and test according to user configuration is done through testng.xml file, user can decide which runs to be excluded or included, which should be 
       		         executed parallel.
       		         How to create a testNG.xml file? right click on project move to testNG->convert to testNG. this will automatically creates a sample testNG file.
      Depends       :User can use this feature to mark a test to be dependent on group or test and decide whether to execute or not based on dependent test.
      Parameters    :User can pass the value from testng file to test using this parameters field.
      Data Provider :This data provider feature helps user test the same test to be executed on multiple data for data testing.
      Reporting     :testng generates the XML and HTML reports.
       

     	TestNG provides the feature of enabling and disabling the test cases. We can disable or enable test cases for executing.
     	We need to provide the include or exclude tag on method level or group level. providing include and exclude inside method tag will work for methods.
     	providing include and exclude inside the group will work for groups.
     	Ex: if we want to test the whatsapp chat feature in regression but we dont want to test the group chats feature, we can user exclude tag and add name of method under
     	    name field.This will execute the whatsapp chat feature excluding the groupChat.
     	
     	<suite name="Regression">  
          <test name="whatsapp chats">  
            <classes>  
                <class name="chats">
                   <methods>  
                      <exclude name = "groupChats"/>  
                   </methods>
                </class>   
            </classes> 
          </test>  
        </suite>
     	
     	

     	Tags : <exclude name = "groupChats"/>  
     	       <include name = "groupChats"/>  
       
       Regex: how to run the test cases using regex.
       TestNG allows to run the test cases by providing the regex pattern to select which test or groups to execute.
       Ex: if we want to execute all the test which has method name ending with chats.
       
     	<suite name="Regression">  
          <test name="whatsapp chats">  
            <classes>  
                <class name="chats">
                   <methods>  
                      <include name = ".*Chats"/>  
                   </methods>
                </class>   
            </classes> 
          </test>  
        </suite>
     	
     	

     	Tags : <exclude name = "personal.*"/>  
     	       <include name = ".*Chats"/>

      @BeforeSuite  : This tag method will get executed before all suite mentioned in xml file. a testng can have multiple suite' 
      @AfterSuite   : This tag method will get executed after all the suite test are executed.
      @BeforeTest   : A suite can have multiple test and this tag method will get executed before each test, so if there are 2 test defined so this tag will get executed 2 times.
      @AfterTest    : This tag method will get invoke after each test is completed.
      @BeforeClass  : This tag method will be executed before each class is started.we can multiple class under test, if there are 3 classes defined then this tag will executed 3 times.
      @AfterClass   : This tag method will get invoke after each class is completed its all test execution.
      @BeforeMethod : This tag method code will be executed before each @test method.
      @AfterMethod  : This tag method code will be executed after each @test method.
      @BeforeGroups : This tag will be executed only once before a group.
      @AfterGroups  : This tag will be executed only once after a group. 
       
      Annotation attribute: 
      description  = @Test(description="This is regression test")
                     user can give the description of the test which can be seen in reports.
      dependsOn    = @Test(dependsOnMethods= {"whatsappSettingProfileTest"})  
                     Provide the method name on which this test is dependent, this will not get executed if the dependent
                     method gets failed.   we can pass multiple method name.
      timeOut      = @Test(timeOut=500) 
      				 Marking the timeout of 500 milisec means the test case will automatically fails if it exceeds 500 milisec. 
      priority     = @Test(priority=1)  
                     When no 'priority' attribute is specified then the TestNG will run the test cases in alphabetical order. 
                     The priority can hold the integer values between -5000 and 5000.
                     If there are 3 test with priority as -2500,0,25 and 80 then the execution sequence will be 0,25,80,-2500.
      enabled      = @Test(enabled=false)  
                     User can mark particular test as disabled with tag enabled as false.this test will not get executed.
      groups       = @Test(groups= {"regression"})  
                     we can add test to a group with any name, we can add test to as many groups we needed with comma separated.                             
       

      We can pass the required from testNG file also with the help of parameters tag feature of testNG.
      In following example we have pass the type of operating system on which whatsapp need to be tested. We need to add name and value as attribute.
      The same name should be used in annotation parameter over the required method as below.
      We can add this parameter tag on suite level or test level. below we have added in suite level means the OS will be same for all test.
      If we have 5 different values we need to add 5 separate parameter tags, but in class method we can add all parameters name with comma separated. 
      
      <suite name="Regression">
       <parameter name="OS" value="IOS" />
        <groups>  
           <run>  
              <include name="smoke"/>  
           </run>  
        </groups>  
          <test name="whatsapp chats">  
            <classes>  
                <class name=".*Test">
                </class>   
            </classes> 
          </test>  
        </suite>
        
        public class whatsapp{
        @Parameters({"OS"})
        public void chatTest(String operatingSystem){
        System.out.println(operatingSystem);                // this will print IOS which is taken from testNG file.
        } }
      
       
       Data providers
       It is used for data testing where user wants to run the functionality with different data like 20 data's or 30 data or more.
       We can add as many data we want. we need to create a method which return the objects containing the data.
       We need to add each and every data in matrix or we can use for loop and retireve the data from testData or any other source and store it in object.
       
       @DataProviders                                 //need to provide tag.
       public Object[][] getData()
       {
       Object[][] data = new Object[3][2];            //here 3 represents rows and 2 represents column  [row][column].
       data[0][0] = "User1";
       data[0][1] = "Password1";
       
       data[1][0] = "User2";
       data[1][1] = "Password2";
       
       data[2][0] = "User3";
       data[2][1] = "Password3";
       returns data;
       }
       
       How to catch the above data in test.
       If the dataProvider is in another class then we needto write in below format.
       @Test(dataProvider = "methodName", dataProviderClass = dataProviderClassName.class)
       
       @Test(dataProviders="getData")              // this is name given as same as above method name given. this method test will run 3 times as number of rows in data is 3.
       public void instaLogin(String user, String pwd)
       {
       clickLogin(user,pwd);
       }
       
       

      Asserts are available in testNG as well in JUnit. both are used for same purpose as to do validation with expected and actual.
      Assertion are of two types: Soft Assertion and Hard Assertion
      But soft assertion is only available with testNG.
      difference between soft and hard assert: Hard assert will stop and execution if it fails whereas soft assert will continue to 
      test next script even after failing. Soft and hard assert provides same method for validation.
      follow below link for assert methods in details:
      Java Doc assert testNG
      How to use soft and hard assert?
      Hard assert:
      import org.testng.Assert;
      	Assert.assertEquals(originalTitle, expectedTitle);
      Soft assert:
      import org.testng.asserts.SoftAssert;
      SoftAssert softassert = new SoftAssert();
      softassert.assertEquals(originalTitle,expectedTitle);
       
       JUnit also provides a assert method with only hard assertion as type.
       how to use?
